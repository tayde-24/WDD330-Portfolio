Validating Forms

With and Without Built-In API Example

Form Exercise 1 Without JavaScript

By putting the word "required" inside the input tag and putting a pattern as well, the browser automatically creates a message if there's no input or if the input does not match the input required. After inputing the correct option and clicking the button, the key-and-value pairs appear in the website.

Form Exercise 2 Using JavaScript

In this exercise, I learned how to create customized error messages. When you don't follow the "type email" form, the customized error shows up. This exercise didn't have a customized error message for when the user doesn't input their email, so I tried doing it with "validity.valueMissing", but it didn't work. I later then discovered that it will only work if the "required" attribute was inside the input tag. I did include it in, but after testing it out, I think this would have confused the user a bit, so I commented it out from the code.

Form Exercise 3 Using JavaScript

In this exercise, this uses the built-in API, so it gives you some sense of control when making these error messages. In the exercise, we made it this input act like it's required. It won't let you submit it blank and gives you a message right away for it.

Form Exercise 4 Using JavaScript

In this exercise, this does not use the built-in API here. Unlike the other form, it's not a required input, so it's made optional. If the user does want to fill it out, the code expects an email and no value else, otherwise, it will give an error message when you try to submit text that's not an email with the use of the Regex pattern value.
