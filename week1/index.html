Intro of Chapter

  • When writing an application you are communicating with the user
  • Any information stored or shared with the user is called an output
  • Anything coming in from where it was stored, or anything the user enters is called input
    • To techies, input and output are called IO, Input-Output

Building the HTML (Review for me)

  • Make sure to give the page tab a title. Also make sure the title makes sense with the content that's being
    presented there
  • What is a placeholder? → A placeholder is a property specific to textarea's and input tags
    • Basically, it can be used to put a hint to your user
  • There are different levels of header tags:

    • h1 → The largest

    • h2 → A little small

    • h3 → Getting smaller

    • h4...h6 → Gets even smaller. You get the idea.

  • Section Tags → Represents an area or chunck of the page when it's displayed in your web browser
  • By adding the "onclick" listener in the button tag, it plays out the function you program it to do
  • Adding CSS can not only make the page work better, but it also makes the web page look pleasing
    to the user's eyes
    • Only when done well that is. Simplicity it the key
  • It doesn't matter what order you put the functions in. JavaScript can handle any order
  • What is var? → Variable
    • Example imagery: It's like a box you can put stuff
    • Make sure it matches the variable; otherwise, it will get confusing and may not work
    • Example: var storyName = document.getElementById("name_input").value
    • The "id" gets the element value under that id name
  • What is local storage? → It stores any information you want to use later.
    • It uses key-value pairs
