Chapter 3 Arrays, Logic, and Loops Notes and Questions

Arrays

  • What is an array? → It's an ordered list of values. It's simply written with a pair of square brackets
    • Ex: const myArray = [];
    • Another way to write is as an array constructor fumction → const myArray = new Array();
      • Though both produce an empty array, it's preferable to stick with the array literals
  • Arrays are not primitive values but a special built-in object
  • Each item in an array can be treated like a variable
  • We can change the value using the assignment operator =
  • How to add values to arrays:
    • const heroes = []; //initializing array
      heroes[0] = "Batman";
      heroes[1] = "Wonder Woman";
      heroes[2] = "Flash";
      heroes[5] = "Aquaman"

      //Result: ["Batman", "Wonder Woman", "Flash", undefined, undefined, "Aquaman"]
  • There's no need to add each value one by one:
    • const avengers = ["Captain America", "Iron Man", "Thor", "Hulk"];

      //Result: ["Captain America", "Iron Man", "Thor", "Hulk"]
  • It doesn't have to be the same types of items inside the array
    • const mixedArray = [null, 1, [], "two", true];
  • Removing values → delete avengers[3]; //Example
    • Whatever value was deleted will contain an undefined values in its place
  • Array Properties and Methods:
    • Finding length of array → avengers.length; // 6
    • Finding the last item of an array → avengers[avengers.length - 1]; //Result: "Hulk"
    • pop() → Removes the last item from an array
    • shift() → Works similar way to pop() method, but removes the first item from an array
    • push() → This method appends a new value to the end of the array
    • unshift() → Similar to push(), but it appends a new item to the beginning of the array
    • concat() → Merges with one or more arrays
    • join() → Turn an array into a string that comprises all the items in the array, separated by commas
    • slice() → Creates a subrray array, chops out a slice of an original array
    • splice() → Removes items from an array then inserts new items in their place
      • This is a DESTRUCTIVE operation as it changes the value of the array
    • reverse() → Reverses the order of an array
    • sort() → Sets the order of an array
      • It changes the order of the array permanently

Refer to book for the rest of the array properties and methods to review.

Sets

  • Sets → Data structure that represents a collection of unique values (it cannot include any duplicate values)
  • They offer a useful way to keep track of data without having to check if any values have been duplicated
    • Quick and easy to check if a particular value is in a set
  • How to create sets → It's created using the new operator and Set() constructor
    • const list = new Set(); //Example
  • Values can be placed into a set using the add() method → list.add(1);
    • Multiple items can be added to the set by repeating the add method → list.add(2).add(3).add(4);
    • Another way to write multiple values → const numbers = new Set([1,2,3]);
    • Type coercion is not used when values are added to a set
  • Set Methods:
    • size() → The number of values in a set
    • has() → Used to check if a value is in a set
    • delete() → Used to remove a value from set (Returns a boolean value of true if the value has been removed
    • clear() → Used to remove all values from a set
    • Array.from() → Makes it possible to convert a set into an array
  • When objects are added to sets, they will be stored there as long as the set exists, even if the original reference to the object
    is removed
  • Weak sets avoid this situation by garbage collecting any references to a 'dead object' that's had its original reference removed

Maps

  • Maps → Another data structure, a convenient way of keeping a list of key and value pairs
    • Similar to 'hashes', or 'hash tables' or 'dictionaries' in other programming languages
  • What makes maps different from JavaScript objects:
    • Maps can use any data type as a key
    • Efficient way to find the number of key-value pairs an object has by using size property
    • Focused on the storage and retrieval of key-value pairs
    • Restricts you to using the get() method to retrieve any values
  • How to create maps → const romanNumerals = new Map();
  • set() Used to add a key and value pair to a map (The first value is the key and the second is the value
  • Map Methods:
    • get() → It helps look up a value
    • has() → Used to check if a particular key is in a map
    • delete() → Used to remove a key and value pair from a map (Returns a boolean)
    • clear() → Removes all key and value pairs from a map

Logic

  • Logical conditions that allow you to control the flow of a program by running different blocks of code,
    depending on the results of certain operations
  • if statement structure:
    • if (condition) {
      // code to run if condition is true
      }
  • else statement structure:
    • if (condition) {
      // code to run if condition is true
      } else {
      //code to run if condition is false
      }
  • Ternary Operator → A shorthand way of writing an if...else statement:
    • condition ? (//code to run if condition is true) : (//code to run if false)
  • Switch statements (makes a logical decision tree):
    • switch (number) {
      case 4:
      console.log("You roled a four");
      break;
      case 5:
      console.log("You roled a five");
      break;
      .
      .
      .
      default:
      console.log("You roled a number less than four");
      break;
      }

Loops

  • Loops will repeat a piece of code over and over again according to certain conditions
  • while loops will repeatedly run a block of code while a certain condition is true, and takes the
    following structure:
    • while (condition) {
      //do something
      }
  • Conditions in a while loop will be met at some point, otherwise your code will get stuck in an
    infinite loop that can possibly crash the program

Refer to book for the rest of the different types of loops to review.
