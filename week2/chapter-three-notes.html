Chapter 3 Arrays, Logic, and Loops Notes and Questions

Comments

  • Writing comments:
    • A piece of code that's ignored by the language
    • Extremely important → Makes things easier for anyone reading your code to understand what's going on
      • Single line comments //
      • Multi-line comments /**/
    • There are utilities to make a document out of your comments

JavaScript Grammar

  • JavaScript known as a C-style syntax because of similarities with C programming language
  • Could be written without semi colons, but it's prone for errors
    • const message = "Hello world!"
      alert(message)

      /*Can get away with it, but it's
      not good practice, especially
      when it comes to complex coding.
      It can cause confusion.*/
    • const message = "Hello world!";
      alert(message);

      //Way better!
  • What is a block? → A series of statements that are collected together inside curley braces
    • Blocks don't need to be terminated by a semi-colon
  • What is whitespace? → Spaces, tabs and new lines that are used to separate different parts of each statement

Primitive Data

  • JavaScript has seven different data types (first six are primitive data types):
    • String
    • Symbol
    • Number
    • Boolean
    • Undefined
    • Null
  • Any value that isn't one of the primitive data is an object
    • Arrays
    • Functions
    • Object literals
  • What does typeof do? → They find out the type of a value
