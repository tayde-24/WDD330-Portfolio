Chapter 4 Functions Notes and Questions

Function

  • Function → A chunk of code that can be referenced by name, and is almost like a small, self-contained mini program
    • Can help reduce repetition
    • Make code easier to follow
    • Considered to be first-class objects → meaning they behave in te same way as all the other primitive data
      types and objects in the language
    • Can be assigned to variables, stored in arrays and can even be returned by another function
  • How to write Function Declarations:
    • function hello() {
      console.log("Hello World");
      }
  • How to write Function Expressions:
    • const goodbye = function() {
      console.log("Goodbye World!");
      };
      //Note that the example ends
      with a semi-colon
    • This is known as an anonymous function
      because it doesn't have a name
  • How to write Function() Constructors:
    • const hi = new Function('console.log("Hi World!");');
    • Not recommended to declare functions this way → numerous problems
      with placing the function body inside a string
    • Functions created this way are also created in the global scope; can lead to
      strange and unexpected behavior

Invoking a Function

  • Invoking a function is to run the code inside the function's body
  • This is how to invoke a function:
    • hello();
      //Result: "Hello World!"
    The function can be invoked over and over again by typing its name followed by parenthesis
  • One advantage of functions is that there's no need to write repetitive blocks of code
  • All the functionality is kept in one place → Known as the DRY principle (Don't Repeat Yourself)

Return Values

  • All functions return a value, which can be specified using the return statement
  • Here's an example that returns a string:
    • function howdy(){
      return "Howdy World!";
      }
    We can now assign a function invocation to a variable, and the value of that variable
    will be the return value of that function

Parameters and Arguments

  • Parameters → Any parameters a function needs are set when the function is defined
  • Arguments → When a function is invoked, it is provided with arguements
    • function square(x){
      return x*x;
      }

      //the x inside the parenthesis
      is a parameter
    • square(4.5); //Result: 20.25
    You can use as many parameters as you like when defining functions
  • If a parameter is not provided as an arguement when the function is invoked, the function will still be invoked,
    but the parameter will be given a value of undefined
  • Default Parameters → These are values that will be used by the function if no arguments are provided when
    invoked
    • function hello(name="World") {
      console.log(`Hello ${name}!);
      }
      //Result without argument: "Hello World!"

      hello("Universe"); //Result: "Hello Universe!"

Callbacks

  • Functions can also be given as a parameter to another function
  • A function that is passed as an arguement to another is known as a callback
    • function sing(song){
      console.log(`I'm singing along to ${song}`);
      }

      sing("Let It Go"); //Result: "I'm singing along to Let It Go"
    We can now assign a function invocation to a variable, and the value of that variable
    will be the return value of that function

Refer to book for the rest of the different callbacks programming to review.

Array Iterators

  • forEach() → This method will loop through the array and invoke a callback function using each value as an argument
    • It takes three parameters:
      • The first represents the value in the array
      • The second represents the current index
      • The third represents the array that the callback is being called on
  • map() → Similar to forEach() method in that it iterates over an array, and takes a callback as a parameter
    • The difference is that it returns a new array that replaces each value with the return value of the callback
  • reduce() → ANother method that iterates each value in the array, but this time it cumulatively combines each result to
    return just a single value.
  • filter() → It returns a new array that only contains items from the original array that return true when passed to the
    callback
