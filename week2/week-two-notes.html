Chapter 2 Programming Basics Notes and Questions

Comments

  • Writing comments:
    • A piece of code that's ignored by the language
    • Extremely important → Makes things easier for anyone reading your code to understand what's going on
      • Single line comments //
      • Multi-line comments /**/
    • There are utilities to make a document out of your comments

JavaScript Grammar

  • JavaScript known as a C-style syntax because of similarities with C programming language
  • Could be written without semi colons, but it's prone for errors
    • const message = "Hello world!"
      alert(message)

      /*Can get away with it, but it's
      not good practice, especially
      when it comes to complex coding.
      It can cause confusion.*/
    • const message = "Hello world!";
      alert(message);

      //Way better!
  • What is a block? → A series of statements that are collected together inside curley braces
    • Blocks don't need to be terminated by a semi-colon
  • What is whitespace? → Spaces, tabs and new lines that are used to separate different parts of each statement

Primitive Data

  • JavaScript has seven different data types (first six are primitive data types):
    • String
    • Symbol
    • Number
    • Boolean
    • Undefined
    • Null
  • Any value that isn't one of the primitive data is an object
    • Arrays
    • Functions
    • Object literals
  • What does typeof do? → They find out the type of a value

Variables

  • What are variables? → They are used in programming language to refer to a value stored in memory
  • const → It's used when the variable will NOT be reassigned to another value
  • let → Used if variable might be reassigned later in the program
  • You can declare and assign multiple variables at the same time IF you separate them with commas
    • Ex: let x = 3, y = 4, z = 5;
  • The only way you can change the value of const is this:
    • const message = {value: "Alexa"} //an object
      name.value = "Siri" //change the value
  • Using const and let to declare variables means they are block scoped, so their value only exists inside the
    block they are declared in
  • Global Scope → Variables that are accessible everywhere in the program
    • Keep it to a minimum though
  • Local Scope → Variables that are inside a block and will only be available inside that block
  • Depending where you put the variables, their values can change sometimes

String Properties and Methods

  • const name = "Alexa";
    name.length     → It tells us how many characters are in the string
    • Another way to write it → name["length"];
  • name.toUpperCase(); → This method writes a string in all capitals
    name.toLowerCase(); → This does the opposite from the other method
  • name.charAt(1); → This method lets you know which character is at a certain position
  • name.indexof("A"); → This method lets you know where a certain character or substring appears in a string
  • name.includes("a"); → This method lets you know if a string contains a certain character
  • name.startsWith("A"); → This method checks if a string starts with a certain character (IT'S CASE SENSATIVE)
    name.endsWith() does something similar in the opposite way
