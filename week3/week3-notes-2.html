Week 03- Notes on DOMs and Events

The Document Object Model

  • The Document Object Model (DOM) allows you to access elements of a web page and enable interaction with a page by adding and removing elements, changing the order, content and attributes of elements, and even altering how they are styled
  • What is a DOM? → It represents an HTML document as a network of connected nodes that form a tree-like structure
    • Treats everything on a web page as a node
      • HTML tags
      • Text inside these tags
      • Even attributes of a tag are all nodes
      HTML tag is the root node, and every other part of the document is a child of this
  • The DOM is not actually part of JavaScript because it is language agnostic → Meaning it can be used in any programming language, not just JavaScript
  • We can use JavaScript to access and modify different parts of a web page using a special built-in object called document
  • Legacy DOM Shortchut Methods
    • Document.body → It returns the body element of a web page
    • Document.images → It returns a node list of all te images contained in the document
    • Document.links → It returns a node list of all the a elements and area elements that have an href attribute
    • Document.anchors → It returns a node list of all the a elements that have a name attribute
    • Document.forms It returns a node list of all the forms in the document
    • NOT ACTUALLY ARRAYS → Node lists are array-like objects, but they are not arrays (you can access each item using index notation)
    • They also have a length property, which can be used to iterate through every element using a for loop
      • for (let i=0; i < document.images.length; i++) {
        // do something with each image using document.images[i]
        }
  • getElementsByTagName() → This returns a live node list of all the elements with the tag name that is provided as an arguement
  • getElementsByClassName() → This returns a live node list of all elements that have the class name that is supplied as an argument
  • document.querySelector() → This allos you to use CSS notation to find the first element in the document that matches a CSS selector provided as an arguement
  • document.querySelectorAll() → Also uses CSS notation but returns a node list of all the elements in the document that match the CSS query selector

Getting and Setting Attributes

  • The DOM has numerous getter and setter methods that can be used to view, add, remove or modify the value of any of these attributes
    • getAttribute() → This method returns the value of the attribute provided as an argument
    • setAttribute() → This can change the value of an element's attribute
      • It takes two arguments: the attribute that you wish to change, and the new value of that attribute
    • className → This allows the class of an element to be set directly
      • Changing the className property of an element by assignment will overwrite all oter classes that have already been set on the element
    • classList → It is a list of all the classes an element has
      • add → This method can be used to add a class to an element without overwriting any classes that already exist
      • remove → This will remove a specific class from an element
      • toggle → A method that will add a class is an element doesn't have it already, and remove the class if it does have it
  • Creating Dynamic Markup
    • createElement() → This method that takes a tag name as a parameter and returns that element
    • document.createTextNode() → This method creates a text node: it takes a parameter, which is a string containing the text that goes in the node
    • A Function to Create Elements
      • function createElement (tag, text) {
        const el = document.createElement(tag);
        el.textContent = text;
        return el
        }
    • Updating CSS
      • Every element node has a style property
      • Any CSS property names that are seperated by dashes must be writtne in camelCase notation, so the dash is removed and the next letter is capitalized because dashes are not legal characters in property names
        • Alternatively, the bracket notation can also be used, meaning that CSS properties can be provided as a string and don't need the camelCase notation

Events

  • Events are another part of the DOM and they are what provides the link between the web page and user interactions
  • Inline-event handlers are best avoided
  • The recommended way of dealing with events, and the current standard, is to use event listeners
  • addEventListener() → This method is called on a node object, the node to which the event listener is being applied
    • The first parameter is the type of event
    • The second is a callback function that is invoked when the even occurs
  • There are event listeners for mouse clicks, key presses, key modifiers, and touch events
  • removeEventListener() → This method removes an event listener
  • preventDefault() → This method can be used inside the callback function to stop the default behavior happening
